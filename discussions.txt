================================================================================
PLAN DÉTAILLÉ ORION CORE — 12 MODULES
================================================================================

MODULE 01 — FOUNDATION (Auth & RBAC)
------------------------------------
Objectif : Socle sécurisé pour dashboard et API.
Contenu :
  - Installation Laravel 10, MySQL, Redis, Horizon, Reverb, Sanctum, Spatie Permission, Spatie Data
  - Login / Logout / Me (Sanctum token)
  - CRUD Users (admin uniquement)
  - Rôles : admin, operator, viewer
  - Permissions granulaires (devices.view, alerts.manage, incidents.assign…)
  - Policies de base
  - Seeder admin par défaut
Fichiers clés :
  - Services/AuthService.php, Services/UserService.php
  - Requests/Auth/LoginRequest.php, Requests/User/StoreUserRequest.php
  - Controllers/Api/V1/AuthController.php, UserController.php

MODULE 02 — DEVICES (Équipements supervisés)
--------------------------------------------
Objectif : Référentiel central de tous les équipements réseau.
Contenu :
  - CRUD équipements (routeur, switch, firewall, imprimante, AP, serveur…)
  - Champs : name, ip, mac, type, vendor, model, firmware, status, discovery_method
  - Liaison device ↔ agent (optionnel, 1 agent = 1 device type server)
  - Filtres et recherche (par type, statut, IP)
  - Enum DeviceType, DeviceStatus
Fichiers clés :
  - Models/Device.php, Services/DeviceService.php
  - Data/DeviceData.php (Spatie Data)
  - Policies/DevicePolicy.php

MODULE 03 — AGENTS (ORION Agent)
--------------------------------
Objectif : Gestion des agents Windows/Ubuntu qui poussent des métriques.
Contenu :
  - Auto-enregistrement agent (POST /agents/register) avec bootstrap key
  - Génération clé API unique (hashée côté serveur)
  - Heartbeat périodique (POST /agents/heartbeat)
  - Middleware AgentApiKeyAuth (routes dédiées agents)
  - Statut online/offline (dernier heartbeat)
  - Job CheckAgentsOfflineJob (Scheduler /1min) → Event AgentWentOffline
Fichiers clés :
  - Services/AgentRegistrationService.php, AgentHeartbeatService.php
  - Middleware/AgentApiKeyAuth.php
  - Jobs/CheckAgentsOfflineJob.php
  - Data/AgentRegisterData.php, HeartbeatData.php

MODULE 04 — METRICS (Métriques & ingestion)
-------------------------------------------
Objectif : Stocker et consulter les métriques temps réel et historiques.
Contenu :
  - Ingestion bulk agent (POST /agents/metrics) — CPU, RAM, disk, temp, network…
  - Ingestion métriques SNMP (via Module 05)
  - Table metrics + table metrics_hourly (agrégats via Job)
  - API consultation : GET /devices/{id}/metrics?type=cpu&from=&to=
  - MetricIngestionService → déclenche évaluation alertes
  - Job AggregateMetricsJob (Scheduler horaire)
Fichiers clés :
  - Services/MetricIngestionService.php
  - Data/MetricBatchData.php, MetricPointData.php (Spatie Data)
  - Jobs/AggregateMetricsJob.php

MODULE 05 — MONITORING (Ping / SNMP / Nmap)
-------------------------------------------
Objectif : Superviser équipements SANS agent.
Contenu :
  - PingService + PingDeviceJob (Scheduler /1min, queue Horizon)
  - SnmpService + PollSnmpJob (uptime, interfaces, bande passante)
  - NmapService + NmapScanJob (découverte réseau, fingerprint)
  - Commande artisan orion:discover {subnet}
  - Mise à jour statut device + création auto si découvert
  - Table interfaces (liées aux devices)
Fichiers clés :
  - Services/PingService.php, SnmpService.php, NmapService.php
  - Jobs/PingDeviceJob.php, PollSnmpJob.php, NmapScanJob.php
  - Scheduler dans app/Console/Kernel.php

MODULE 06 — ALERTING (Alertes intelligentes)
----------------------------------------------
Objectif : Détecter et notifier les seuils critiques.
Contenu :
  - CRUD AlertRule (CPU>90%, RAM>90%, disk>95%, temp critique, device offline…)
  - AlertEvaluator (appelé après chaque ingestion métrique)
  - Modèle Alert (raised, acknowledged, resolved)
  - Event AlertRaised → Listener(s) notification + broadcast Reverb
  - API : liste alertes, acquitter, résoudre
Fichiers clés :
  - Services/AlertEvaluator.php, AlertService.php
  - Events/AlertRaised.php
  - Listeners/BroadcastAlertListener.php, LogAlertListener.php
  - Enums/AlertSeverity.php, AlertStatus.php

MODULE 07 — INCIDENTS (Gestion des incidents)
---------------------------------------------
Objectif : Cycle de vie des incidents liés aux alertes.
Contenu :
  - CRUD incidents
  - Machine à états : Ouvert → Assigné → En cours → Résolu → Fermé
  - Assignation à un utilisateur (operator/admin)
  - Priorité (low, medium, high, critical)
  - Liaison alerte → incident (escalade auto ou manuelle)
  - Event IncidentUpdated → broadcast Reverb
Fichiers clés :
  - Services/IncidentService.php
  - Data/IncidentTransitionData.php
  - Events/IncidentUpdated.php
  - Enums/IncidentStatus.php, IncidentPriority.php

MODULE 08 — TOPOLOGY (Cartographie réseau)
------------------------------------------
Objectif : Fournir les données pour Cytoscape.js (dashboard).
Contenu :
  - Table topology_links (source_device, target_device, link_status)
  - Construction auto des liens (LLDP SNMP, découverte Nmap, sous-réseaux)
  - API GET /topology → JSON nodes + edges (format Cytoscape)
  - Mise à jour statut liens en temps réel (via Events)
Fichiers clés :
  - Services/TopologyService.php
  - Data/TopologyNodeData.php, TopologyEdgeData.php

MODULE 09 — REALTIME (Laravel Reverb)
-------------------------------------
Objectif : Pousser les mises à jour live au dashboard React.
Contenu :
  - Config Reverb + Laravel Echo côté doc
  - Canaux : private-org.alerts, private-device.{id}.metrics, presence-agents
  - Events ShouldBroadcast : AlertRaised, MetricReceived, AgentStatusChanged,
    IncidentUpdated, DeviceDiscovered, AnomalyDetected
  - routes/channels.php avec autorisation Sanctum
Fichiers clés :
  - Events/* (implements ShouldBroadcast)
  - config/reverb.php, config/broadcasting.php

MODULE 10 — ORION AI (OpenRouter)
---------------------------------
Objectif : Anomalies, prédictions, assistant administrateur.
Contenu :
  - Client OpenRouter (clé serveur .env uniquement)
  - AnomalyDetector (Job DetectAnomaliesJob /15min) — pics, comportements suspects
  - PredictiveAnalyzer (Job RunPredictionJob quotidien) — saturation disque/RAM
  - OrionAssistant (POST /ai/ask) — questions en langage naturel
  - Contexte IA : métriques récentes + alertes + incidents + logs
  - Table anomalies (device_id, type, score, context JSON)
Fichiers clés :
  - Services/AI/OpenRouterClient.php, AnomalyDetector.php, PredictiveAnalyzer.php, OrionAssistant.php
  - Jobs/DetectAnomaliesJob.php, RunPredictionJob.php
  - Data/AiAskData.php, AnomalyResultData.php

MODULE 11 — REPORTS (Rapports)
------------------------------
Objectif : Rapports périodiques PDF/JSON.
Contenu :
  - Rapports quotidiens, hebdomadaires, mensuels
  - Contenu : disponibilité, alertes, incidents, anomalies, analyse IA
  - Job GenerateReportJob (Scheduler cron)
  - Stockage fichier + API GET /reports
  - Export PDF (DomPDF ou Spatie PDF)
Fichiers clés :
  - Services/ReportGenerator.php
  - Jobs/GenerateReportJob.php
  - Data/ReportContentData.php

MODULE 12 — DASHBOARD API (Vue globale)
---------------------------------------
Objectif : Endpoints agrégés pour le tableau de bord principal.
Contenu :
  - GET /dashboard/overview : résumé réseau (devices count, agents online/offline,
    alertes actives, incidents ouverts, santé globale %)
  - GET /dashboard/health : score santé réseau calculé
  - Cache Redis (TTL court) pour performances
Fichiers clés :
  - Services/DashboardService.php
  - Controllers/Api/V1/DashboardController.php
